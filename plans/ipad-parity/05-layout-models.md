## Current parity

Area 6 (Model Creation & Management): 68 features — **35 implemented, 13 partial, 20 missing, 0 out-of-scope**. Area 7 (Model Types & Properties): 58 features — **41 implemented, 8 partial, 8 missing, 1 out-of-scope**. Combined this is the single most complete iPad subsystem: `LayoutEditorView.swift` (~12.5K lines) plus ~30 Phase-J bridge methods already deliver model creation for 16 types, .xmodel import (single + multi), vendor download, ARKit Map-from-Lights, full tap/drag/resize/rotate editing, multi-select align/distribute/match/flip/duplicate/group, the complete per-type + controller-connection property editor (every 8 DMX subtype editable), layout undo, group CRUD, view objects, preview switching, and 2D/3D toggle. The remaining work is overwhelmingly **SwiftUI + ObjC++ bridge wiring over already-shared `src-core/` logic** — the model algorithms, serializers, channel-layout HTML, and CAD/export functions already live in core; only CAD export and layout-image printing have weak iOS justification.

## Approach

- **Reuse shared core, don't reimplement.** Confirmed core surfaces this theme builds on: `XmlSerializer::SerializeModel()/SerializeModels()` (model copy/paste + .xmodel export), `Model::ExportAsCustomXModel()/ExportAsCustomXModel3D()` (export-as-custom), `Model::ChannelLayoutHtml()` (Node Layout view), `ModelManager::CreateDefaultModel()` + `CreateDefaultModelFromSavedModelNode()` (already driven by `createModelOfType:`/`importXmodelFromPath:`), `SubModel` range ops, and the existing `recalcModelStartChannels` rework path. Each work item below names the exact core call.
- **Bridge convention.** New ObjC++ methods land on `XLSequenceDocument` (or `XLMetalBridge` for canvas/geometry ops) following the `NS_SWIFT_NAME(…)` style already in `XLSequenceDocument.h`. All mutations funnel through `SequencerViewModel` so the existing undo-snapshot (`pushLayoutUndoSnapshot*`) + row-reload happen in one place.
- **Touch idioms over literal ports.** Desktop right-click submenus become long-press menus / inspector sections; the wx grid dialogs (Custom Model, Node-Select) become Metal/Canvas-backed touch grids reusing the existing `CustomModelEditorSheet` canvas and `iPadModelPreview` node hit-testing. Clipboard ops bridge to `UIPasteboard`; file export uses `fileExporter`.
- **The Node-Select grid is the keystone.** Faces/States/SubModels all degrade to raw attribute editors today purely because there's no touch node-picker. Building one `NodeSelectGridView` (Phase 1) is the highest-leverage item — it lifts three Area-7/8 partials toward parity at once.

## Out of scope for this theme

## Risks / open questions

- **`LAY-2` Node-Select grid is on the critical path** for `LAY-3/4/5/34`. Scope it deliberately to reuse the `CustomModelEditorSheet` canvas renderer and `iPadModelPreview` node hit-testing rather than building a new renderer; over-scoping it stalls three downstream items.
- **`LAY-7` Wiring Diagram** and **`LAY-24` Replace Model** both touch logic currently embedded in wx (`WiringDialog.cpp`, `LayoutPanel.cpp`). Decide early per item: extract the wx-free computation into `src-core/` (preferred, benefits both platforms and satisfies the core-boundary rule) vs. recompute in the bridge. The pixel-path geometry for wiring is the candidate for core extraction.
- **Cross-sequence model paste (`LAY-8`)** must validate start-channel/controller-assignment on paste (the pasted model may reference a controller that doesn't exist in the target show) — reuse the `recalcModelStartChannels` + chain-fixup path and surface a warning rather than silently producing invalid channels.
- **Custom-model transforms (`LAY-20`)** must round-trip cleanly through `setCustomModelData` and trigger an `AbortRender` + recalc; verify the grid-string format (`,`/`;` row/layer delimiters) is preserved by the bridge before shipping destructive ops like trim/compress.
- **Bulk-edit silent no-ops (`LAY-10`):** propagating a type-specific property to a mismatched-type model currently no-ops silently; confirm that's the desired UX or surface a "applied to N of M" toast so users aren't surprised.
- **iPad low-def render** now defaults **off** (`IsLowDefinitionRender()` reads the opt-in `render.lowDefinition` pref, default OFF = full-def), so node grids reflect full-def node geometry by default. Still confirm the Node-Select grid and Node Layout view use the model's logical node map, not the render buffer — so that if a user *does* opt into low-def, the node editors are unaffected.
- **Roster multi-select migration (`LAY-41`)** changes `modelListBinding` from `Binding<String?>` to a `Set<String>` and must keep the canvas and roster reading the *same* selection state — diverging them would let the action bar / bulk-edit operate on a stale set. Gate the multi-select strictly behind `EditMode=.active` so single-tap navigation still drives the canvas as today.
- **Accessibility is cross-cutting (`LAY-41`b):** the canvas-gesture-only editing gap also affects the sequencer effect grid (doc 01). The VoiceOver-navigable roster+inspector path should be designed once and shared, not bolted onto layout alone.
- **`LAY-36` is trigger-gated, not speculative:** don't build vendor auth until a catalog actually gates content behind login — App Store review may also scrutinize stored credentials, so prefer cookie/`URLSessionConfiguration` over persisting passwords.
