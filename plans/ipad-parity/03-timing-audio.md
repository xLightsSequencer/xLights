# 03 ¬∑ Timing Tracks & Audio

> **Status:** This theme is **near full parity**. The iPad app implements the entire waveform analysis surface: every filter band (Raw/Bass/Treble/Alto/Non-Vocals/Custom/LUFS/Vocals), on-device HTDemucs stem separation, SoundAnalysis classification, spectrogram view, pitch contour, onset overlay, double-height toggle, and an alternate-audio-track switcher ‚Äî all in `SequencerGridV2View`'s waveform `confirmationDialog`. Timing-track creation (`AddTimingTrackSheet`) covers Empty / fixed 25-50-100 / custom-interval / metronome (+random +tags) / FPP Commands / FPP Effects / Audio Onsets / Tempo / Chords / AI Lyrics / LRCLIB ‚Äî matching the desktop `NewTimingDialog`. Mark operations (add, split-at-playhead, merge-with-next, move/resize-edge drag, delete, rename, breakdown phrases/words, remove words/phonemes, -shimmer toggle, auto-label, halve, divide, subdivide, select-all) are all present on iPad, and almost every one has a desktop equivalent (the desktop mark ops live in `EffectsGrid::rightClick`, not just `RowHeading`). Playback speed (all 8 rates), volume, tags (set/go-to/clear), zoom-to-fit/selection, and hide/show-all-timing reach parity. The **real iPad gaps** are: VAMP Queen-Mary plugins (no FFmpeg/VAMP host on iPad), the **Polyphonic-Transcription** Import-Notes source (VAMP ‚Äî the MIDI / MusicXML / Audacity sources now land on iPad via the wx-free `NoteImporter`), **Prepare Audio** (Reaper/xAudio), and several **preferences** (alternate timing format, small waveform, exclude-audio-from-package; snap-to-timing-marks and timing-play-on-double-click toggles are now at parity). Audio export/encode, Papagayo `.pgo` export, and waveform range-drag selection all reached parity (see scorecard). A couple of iPad touch idioms (double-tap-mark to loop-play, Pencil-precision edge grab) have no desktop equivalent but are additive, not gaps.

## iPad gaps (desktop has, iPad missing)

### P2

- **Shift Effects And Timing / Shift Effects / Shift Selected Effects** ‚Äî *(landed)* desktop `Menu3` items (`xLightsMain.cpp:1088-1092`, `ShiftEffectsOnLayer`/`ShiftSelectedEffectsOnLayer` `:5970,5978`) now have iPad equivalents. Three Edit-menu entries (`XLightsCommands.swift:106-124`) open a shared `ShiftEffectsSheet.swift` with a signed-ms field (frame stepper + frames readout) and an All Effects / All Effects And Timing / Selected Effects Only scope picker (the last gated on a selection). Bridge methods `XLSequenceDocument.shiftAllEffects(byMS:includeTiming:)` and `shiftSelectedEffects(byMS:atRows:effectIndices:)` walk the master-view elements (top layers + strand node layers + submodel layers + timing-track marks when `includeTiming`) exactly like desktop, rounding to the frame interval and applying the identical 0-boundary clamp (truncate to 0 / delete below 0; selected-scope rejects clashes with unselected effects). Undo is provided by a bridge-side per-layer snapshot token (`restoreShiftSnapshot:`) restored on `Cmd+Z` ‚Äî an improvement over desktop, which warns the op can't be undone. The op clears selection, reloads rows, and kicks a full re-render via `SequencerViewModel.shiftEffects(byMS:scope:)`.
- **Snap-to-timing-marks preference** ‚Äî ‚úÖ **Done.** The `snapToTimingMarks` @AppStorage (default ON via absent-key check) gates `EffectsMetalGridView.snapMS` at every drag/resize call site; a "Snap to Timing Marks" toggle lives in the top-chrome options menu (`SequencerGridV2View.swift`). `TimingEffectsMetalGridView` does not snap marks to other marks, so no change there.
- **Timing play-on-double-click toggle** ‚Äî ‚úÖ **Done.** The `timingPlayOnDoubleTap` @AppStorage (default ON) gates `SequencerViewModel.playLoopForTimingMark`; the toggle sits next to the snap toggle in the top-chrome options menu. When off, double-tapping a mark no longer starts loop playback.
- ~~**Drag-select a waveform/timeline range**~~ ‚Äî ‚úÖ **Done.** A two-finger pan on the waveform strip (`TopChromeMetalGridView.onRangePan`) sweeps out the loop region via `onSetLoop`, the iPad's selection equivalent that drives Render Selected Region + Zoom to Selection. The single-finger scroll pan is capped at one touch so tap-to-seek and horizontal scroll are unaffected. Chosen idiom: two-finger (rather than long-press-drag, which is already taken by the ruler loop-set + waveform filter menu).
