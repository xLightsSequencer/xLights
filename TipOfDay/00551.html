Alternate Nodes On Matrices And Trees

Matrix and tree models support an alternate nodes option for both horizontal and vertical layouts. It tells xLights that your strands are wired so that every other node is used.

This matches a common building technique where lights are folded back on themselves, so the physical spacing ends up doubled compared to a plain run. Turning on alternate nodes makes the model in xLights reflect how the prop is really laid out.

Getting this setting right means effects map onto the display the way you expect, instead of looking stretched or offset because the node positions did not match the wiring.
