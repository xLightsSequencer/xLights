Assign Models To No Controller

When you create a model you can leave it set to No Controller instead of immediately picking a controller and start channel. This is the default for new models.

A model on No Controller still works for sequencing and previewing, but it does not claim any channels. That avoids the common headache of new models accidentally overlapping the channels of props you have already set up.

When you are ready, assign the model to a real controller and port. Until then it stays out of the way, which keeps your channel layout clean while you are still building your display.
