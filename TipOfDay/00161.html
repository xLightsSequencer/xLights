Move Nodes In A Single Submodel

There is now an action to move the nodes that belong to a single submodel.

When you are editing submodels, you can shift just that submodel's nodes without disturbing the rest of the model. This makes it much easier to correct a submodel whose nodes ended up in the wrong place or need to be nudged into a better arrangement.

If you build custom models with several submodels, this saves you from rebuilding a submodel from scratch just to fix its node positions.
